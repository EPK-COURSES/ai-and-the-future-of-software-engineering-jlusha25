# Personal Reflection — Topic 8: Real Industry Case Studies

When I first started this assignment, I honestly thought that AI tools like GitHub Copilot or ChatGPT were mostly used by students or junior developers just to quickly look up code fragments or fix simple errors. However, doing this research on Microsoft and Google really changed my perspective. I was surprised to see how deeply these big tech companies have integrated specialized AI models into their actual daily engineering workflows, using them to automate pull request summaries and find architectural defects before a human even reviews the code. While this level of automation saves a lot of time, it also makes me worry about the future. I am concerned that software engineers might become too dependent on these tools and stop thinking critically, which can lead to a lot of technical debt if they just accept code suggestions blindly without fully understanding them. Also, since AI can make mistakes and hallucinate, human code verification is more critical than ever to keep software safe. On the positive side, I see a great opportunity for developers to offload boring, repetitive tasks like writing boilerplate code or setting up basic tests. This shifts our value up the pipeline, leaving us more time to focus on complex logical problems and high-level system design. As a student preparing to enter the job market, I realize that I cannot just focus on learning basic syntax, since AI can write code much faster than me. I need to focus on what AI cannot do, which is strong logic, critical thinking, and system architecture. I plan to use AI tools to speed up my learning, but I have set a strict personal rule for myself: I will never push or commit any AI-generated code to GitHub if I do not completely understand it myself and if I am not ready to defend or fix it manually.
